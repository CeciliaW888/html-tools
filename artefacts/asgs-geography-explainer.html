How Australia is cut into areas — and who cuts it

The ABS builds one tidy nesting hierarchy (Mesh Block → SA1 → … → Australia) and then approximates everyone else's boundaries — postcodes, councils, electorates, suburbs — on top of it. That's why a postcode never lines up perfectly with statistical data. Click around to see how each unit compares.

The Main Structure — one clean hierarchy

Each level is built entirely from the one below, with no overlaps and no gaps. Every Mesh Block sits in exactly one SA1, every SA1 in exactly one SA2, and so on. Click a level for detail. Bars show typical population on a log scale.

The key idea

The SA1 is the workhorse: it's the smallest area with full census data, and it's where SEIFA (incl. IRSAD) and Remoteness are actually computed. Everything coarser — suburb, postcode, LGA — is a population-weighted roll-up of SA1s. The Mesh Block is smaller still, but only releases dwelling/person counts (privacy), so it's used as the building block, not for analysis.

Non-ABS structures — borrowed boundaries

These belong to other authorities (Australia Post, councils, electoral commissions, state gazetteers). The ABS re-creates them by gluing together whole Mesh Blocks that best fit — so they're approximations that don't nest cleanly into the main structure.

⚠️

Why "postcode → anything" is fuzzy

A POA (the ABS's postcode approximation) can straddle several SA2s, cross an LGA line, even span a state border. So when you map a postcode to SEIFA, remoteness or an LGA, you're really asking "which value holds the majority of this postcode's people?" — and sometimes there's no clean winner.

Remoteness Areas — 5 classes

Built from SA1s using the ARIA+ road-distance-to-services score. This is the structure you asked about for postcode→remoteness.

SEIFA — 4 socio-economic indexes

All computed at SA1 from ~16 census variables, then aggregated up. Higher = more advantaged (except IRSD, which measures disadvantage only). ~1000 ≈ national median.

IRSD ≠ IRSAD

A common mix-up: IRSD ranks disadvantage; IRSAD blends advantage and disadvantage. They give different rankings for the same suburb — so always check which one a source quotes.

ABS vs the original authorities

For most "everyday" boundaries there are two versions: the legal/official one from its owner, and the ABS's Mesh-Block approximation used for statistics. They're close, not identical.

Postcodes aren't really areas

Australia Post postcodes are delivery constructs — a postcode can be a single large building, a PO-box range, or a roaming rural round, with no official polygon. The ABS POA invents an areal footprint so postcodes can be mapped. That's why "the postcode boundary" you see is usually really an ABS POA.
