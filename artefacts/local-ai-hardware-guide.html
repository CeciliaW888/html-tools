01 The concepts that matter

Two numbers govern everything. Memory capacity — on an NVIDIA card, or on a Mac or Strix Halo box — decides which models you can load at all. Memory bandwidth decides how fast they generate text.

How much memory a model needs

Rule of thumb: VRAM (GB) ≈ params (billions) × bytes-per-param × 1.2, where bytes-per-param is 2 for FP16, 1 for Q8, and ~0.5 for Q4. A 7B model needs ~4–6GB at Q4, a 32B needs ~20GB, a dense 70B needs ~40GB+. Long contexts add more, because the grows with the conversation.

— the lever that fits big models on small cards

Q4_K_M is the default sweet spot: about half the size of FP16 for only a 3–5% quality drop. Step up to Q5 or Q8 (near-lossless) for coding and reasoning if you have headroom; reserve FP16 for fine-tuning or maximum fidelity.

— what the speed numbers mean

Roughly: 5 tok/s is slow but usable (good for overnight batches), 10–15 tok/s reads comfortably, 30+ tok/s feels instant. Text generation is almost entirely bandwidth-bound — it depends on how many bytes of weights are read per token, not raw compute. That's why a 5090 feels instant and a unified-memory box feels deliberate.

The MoE trick. Modern Mixture-of-Experts models (gpt-oss 120B, Qwen3 235B, Qwen3-30B-A3B) keep all weights resident but fire only a few billion per token — gpt-oss 120B uses ~5B of ~117B each step. So a slow-bandwidth box can run a 120B MoE at usable speed yet crawl on a dense 70B. On any unified-memory machine, prefer MoE models.

07 The upgradeability question

Only a full ATX tower lets you add GPUs later. If future expansion is the goal, your base build needs:

  • PSU: 1000W+ for a single 5090; 1500–1600W if you plan dual high-end GPUs.
  • Motherboard: ATX with two physically spaced PCIe x16 slots so a second triple-slot card fits and breathes.
  • CPU/platform: a Ryzen 9 or Core Ultra is fine for two GPUs; Threadripper for 3–4 or lots of PCIe lanes.
  • Case + cooling + power circuit: high-airflow case and wall-circuit headroom (dual 5090s can pull >1kW).
Portability and upgradeability are opposites. Mini-PCs (Strix Halo, DGX Spark) and Macs are sealed — soldered memory, no internal GPU expansion. Some Strix Halo boxes add for an external GPU, but only at PCIe 4.0 x4. You can't have both — decide which matters more before you spend.

08 Software & models

Ecosystem — NVIDIA is the easy path

CUDA just works. Ollama, LM Studio, llama.cpp, vLLM, TensorRT-LLM — all target NVIDIA first, on Windows and Linux. AMD ROCm has matured (v7.x, working Ollama/llama.cpp/vLLM on Linux) but Windows is still fragile; the RX 7900 XTX (24GB, ~AU$1,200+) is the value AMD card only if you're on Linux and happy to debug. Apple Metal/MLX works well via llama.cpp and MLX.

Easy runners

  • LM Studio — polished GUI, model browser, click-to-run. Best for getting started and quick experiments.
  • Ollama — command line plus an OpenAI-compatible API. Best for coding agents and automation. Both use llama.cpp underneath, so speed is near-identical; many people run both.

Models for your use cases

For general experimentation, coding agents, and especially bilingual EN/ZH content for RedNote, the Qwen family (Alibaba, Apache 2.0) is the strongest all-rounder — Chinese-first and excellent English, from 0.6B edge models to 235B MoE flagships, with dedicated Qwen Coder variants. Also strong: DeepSeek (reasoning/coding), Gemma (efficient), gpt-oss (fast MoE), Llama, Mistral.

09 The staged recommendation

Stage 1 — Best value entry

A used RTX 3090 (24GB) in an expansion-ready ATX tower (~AU$2,500–3,000). Run Qwen3 32B at Q4 and learn what you actually need. Move on when you're consistently waiting on 32B output or hitting the 24GB ceiling.

Stage 2 — Best balance for you ★

A full ATX tower around a single RTX 5090 (32GB), 1200–1600W PSU, two spaced PCIe slots, Ryzen 9 CPU (~AU$7,500–9,000 complete). Runs everything up to 32B instantly and big MoE models very well — then add a second GPU later for 56–64GB. This nails all three of your priorities: PC, customizable, upgradeable.

Stage 3 — Only if you must run dense 70B+ in one box

Either dual RTX 3090/5090 (stays upgradeable) or a sealed 96GB RTX PRO 6000 workstation (~AU$18,000+). Confirm you truly need dense 70B rather than an MoE alternative first — a single 5090 already handles those.

The portability fork: if travelling with the machine matters most, a Strix Halo 128GB mini-PC (~AU$3,000 imported / ~AU$4,750 local) is the best travelable PC for big-but-slow models — just accept it's sealed and slow on dense models.

When to skip all this: a Mac Studio suits you only if you want near-silence and big unified memory without CUDA. Cloud rental (~AU$1.20/hr for a 5090) wins if your use is occasional — buying only pays off after ~700–1,400 active hours, so rent first if you'll run models just a few times a week.

10 Caveats

  • Prices are unusually volatile. A DRAM/GDDR shortage through late 2025–2026 pushed prices sharply up — the RTX 5090 entry price roughly doubled from its ~US$1,999 Nov 2025 low to over US$3,000 by Feb 2026, and some consumer memory rose up to ~89% in Q2 2026. Verify live pricing before buying.
  • Imported mini-PCs (GMKtec, Beelink, Framework) may ship from the US, bill in USD, and lack local warranty — budget 10% GST, shipping, and RMA-to-China risk.
  • All tok/s figures are approximate and shift with quantization, context length, runtime (llama.cpp vs vLLM vs SGLang), and drivers. Treat the relative ordering as the durable signal.
  • Used GPU prices vary by condition — inspect, stress-test, or buy from sellers offering a warranty.

11 Glossary

VRAM capacity
Video RAM — the fast memory physically attached to a GPU. A model's weights must fit in VRAM (or unified memory) or speed collapses. The most important spec for deciding which models you can run.
Unified memory capacity
One large pool shared between CPU and GPU, used by Apple Silicon and AMD Strix Halo. Gives huge capacity cheaply (128GB+) but at lower bandwidth than dedicated VRAM, so big dense models load but run slowly.
Memory bandwidth speed
How many bytes of memory the chip reads per second (GB/s). Because text generation is bandwidth-bound, this number basically is your tokens/sec ceiling. 1,792 GB/s feels instant; 256 GB/s feels deliberate.
Throughput / tokens per second speed
How fast the model produces text. A token is roughly a word-piece. ~5 = slow but usable; 10–15 = comfortable reading speed; 30+ = instant and interactive.
Quantization (Q4, Q5, Q8, FP16) capacity
Compressing weights to lower numeric precision so a bigger model fits in less memory. Q4_K_M ≈ 0.5 bytes/param (sweet spot, ~3–5% quality loss); Q8 ≈ 1 byte (near-lossless); FP16 = 2 bytes (full precision, for fine-tuning).
Parameters (B) model size
The number of learned weights, in billions. Bigger generally means smarter but needs more memory. The first thing you check against your VRAM budget.
Dense model architecture
Activates all its parameters for every token. A dense 70B genuinely needs ~40GB+ and full bandwidth — the hardest case for slow-memory boxes.
MoE / Mixture-of-Experts architecture
Holds all weights in memory but activates only a few expert sub-networks per token (gpt-oss 120B fires ~5B of ~117B). Runs far faster than its size suggests — the key to making unified-memory boxes useful.
KV cache memory
The working memory a model uses to track the current conversation. Grows with context length and can rival the weights at 32K+ tokens — extra VRAM to budget for long prompts.
Context window memory
How much text (in tokens) the model can consider at once. Longer windows are powerful but consume more memory via the KV cache.
CUDA software
NVIDIA's GPU compute platform. Every major local-AI tool targets it first, which is why NVIDIA cards "just work" and are the path of least resistance.
ROCm software
AMD's answer to CUDA. Much improved (v7.x) and solid on Linux, but Windows support is still fragile — usable if you're comfortable debugging.
Metal / MLX software
Apple's GPU framework (Metal) and ML library (MLX). Run local models well on Mac via llama.cpp and MLX.
Ollama runner
A command-line tool with an OpenAI-compatible API for running local models. Best for developers, coding agents, and automation.
LM Studio runner
A polished desktop GUI with a model browser and click-to-run. Best for getting started and quick experiments. Uses llama.cpp under the hood, like Ollama.
llama.cpp engine
The open-source inference engine that powers both Ollama and LM Studio — the reason their token speeds are near-identical.
vLLM engine
A high-throughput serving engine optimised for many simultaneous requests — used more in production than single-user desktop chat.
ATX tower form factor
A standard full-size desktop case and motherboard. The only form factor that lets you add or swap GPUs over time — the upgradeable choice.
PCIe slot / lanes hardware
The high-speed connectors GPUs plug into. For multi-GPU you want two physically spaced x16 slots and a platform with enough lanes to feed them.
PSU hardware
Power supply unit. A single 5090 wants 1000W+; dual high-end GPUs want 1500–1600W. Size up front if you plan to expand.
OCuLink hardware
A port on some mini-PCs for attaching an external GPU — but only at PCIe 4.0 x4 bandwidth, far slower than an internal slot. A compromised upgrade path for otherwise-sealed boxes.
Strix Halo / Ryzen AI Max+ 395 product
AMD's high-end APU with up to 128GB unified memory. Powers small, quiet mini-PCs that can load huge models — but at low bandwidth, so best with MoE models.
DGX Spark product
NVIDIA's compact 128GB unified-memory developer kit (GB10 chip). Mirrors NVIDIA's datacenter software stack but is bandwidth-limited (273 GB/s) — a dev tool more than a value inference box.
